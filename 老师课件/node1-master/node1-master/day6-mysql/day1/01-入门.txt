MySQL 数据模型：
    关系型数据库：一张张类似 excel 的二维表， 组成的数据库。
    优点：
        1.都是表结构，格式统一，易于维护。
        2.可以使用通用的 SQL 语言操作，使用方便，可用于复杂查询
        3.数据存储在磁盘中，安全。
    层级结构：客户端通过SQL ————> 数据库管理系统(MySQL)  ————> 数据库 ————> 数据表  ————> 数据(行、列)
    MySQL的目录结构：
        bin: 可执行文件。(配置环境变量，供全局使用)
        data: 创建的数据库，对应的文件保存路径。
        docs：文档
        include: C 语言的库文件
        lib：C 语言的库文件
SQL 简介：
    英文：Structured Query Language
    结构化查询语言，一门操作关系型数据库的编程语言。
    定义操作所有关系型数据的统一标准。
    对于同一个需求，每个数据库操作的方式可能存在差异，我们成为方言。
SQL 通用语法：
    1.SQL 语句可以单行或多行书写，以分号结尾。
    2.MySQL 的 SQL 语言不区分大小写，关键字建议大写。
    3.注释：
        单行注释：
            -- 注释内容(有一个空格)
            #注释内容(没有空格，MySQL独有)
        多行注释：/*注释内容*/
系统自带的数据库：
    information_schema
        保存了MySQl服务所有数据库的信息。
        具体MySQL服务有多少个数据库，各个数据库有哪些表，各个表中的字段是什么数据类型，
        各个表中有哪些索引，各个数据库要什么权限才能访问。
    performance_schema
        主要用于收集数据库服务器性能参数
        提供进程等待的详细信息，包括锁、互斥变量、文件信息；
        保存历史的事件汇总信息，为提供MySQL服务器性能做出详细的判断；
    mysql
        保存MySQL的权限、参数、对象和状态信息。
        如哪些user可以访问这个数据、DB参数、插件、主从
    sys
        Sys库所有的数据源来自：performance_schema。
        目标是把performance_schema的把复杂度降低，让DBA能更好的阅读这个库里的内容。
        让DBA更快的了解DB的运行情况。
